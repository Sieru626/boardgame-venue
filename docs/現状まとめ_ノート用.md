## 2. ディーラーちゃん（AI）の現状

### 2.1 役割

- サイバーカジノの専属ディーラー「ディーラーちゃん」として振る舞う。
- **Phase 1**: プレイヤーとの自然なチャット（Gemini API）。
- **Phase 2**: 「〇〇の会場作って」などに応じて、**ゲームモードを実際に切り替える**（システム操作の代行）。

### 2.2 キャラ設定（プロンプト）

- ドジっ子の新人アルバイト。自信なさげでオドオド。
- 「えっと…」「あわわ…」「〜っす！」などの口調を指示。
- 「AIですので」「スリリングな対話を楽しむのが仕事よ」などは禁止。

### 2.3 会場作成（Phase 2）の流れ

1. ユーザーが「ミックスジュースの会場作ってー」「セッティングして」などと発言。
2. クライアントが `/api/ai/chat` に送信。サーバーが Gemini を呼び出し、`actionCommand: { type: "change_mode", gameMode: "mixjuice" }` などを返す想定。
3. **サーバー側フォールバック**: AI が `change_mode` を返さなくても、ユーザー発言からキーワード検知して `actionCommand` を上書き。
   - 例: 「ミックスジュース」「会場作って」「セッティング」→ `gameMode: "mixjuice"` など。
4. クライアントが `actionCommand.type === 'change_mode'` かつホストの場合、`socket.emit('host_action', { type: 'change_mode', payload: { gameMode } })` を発火。
5. サーバーが `host_action` で `GameTemplate` を取得し、`selectedMode` 更新・`broadcastState` で全員に反映。画面の GAME SELECT が切り替わる。

### 2.4 対応ゲームモード（会場作成）

| 発言例 | gameMode | 備考 |
|--------|----------|------|
| ミックスジュースの会場、セッティング、準備して など | `mixjuice` | デフォルトでも使用 |
| ババ抜き、ババぬき | `oldmaid` | —
| 神経衰弱 | `memory` | —

### 2.5 UI 表示

- **思考中**: API 呼び出し中。セリフ欄に「ディーラーちゃん思考中…」。最低 700ms 表示。
- **作業中**: `change_mode` 実行中。セリフ欄に「ディーラーちゃん作業中…（会場を準備しています）」＋チャットに「えっと、会場を準備しています…！」を送信。

### 2.6 主要コード（AI・会場）

- **サーバー**: `server/index.js`
  - `/api/ai/chat`: プロンプト構築、Gemini 呼び出し、**会場作成フォールバック**（キーワード検知）、`actionCommand` 返却。
  - `host_action` の `change_mode`: `payload.gameMode` でテンプレート検索、`initializeStateFromActiveTemplate` で状態更新、`broadcastState` で配信。
- **クライアント**: `client/app/room/[id]/page.tsx`
  - `sendDealerMessage`: `/api/ai/chat` 呼び出し、`actionCommand` に応じて `dealerWorking` を立てて `host_action` 発火。
- **パネル**: `client/app/components/AIDealerPanel.tsx` — `isThinking` / `isWorking` で思考中・作業中を表示。

## 3. ゲームモード・テンプレート

- **DB**: `GameTemplate` に `mode`（例: mixjuice, oldmaid, memory）とテンプレート情報を保存。
- **手動適用**: ゲームライブラリから `apply_game_template`（templateId）で会場に適用可能。
- **AI 経由**: 上記のとおり `change_mode` で `mode` 指定のみ対応。**オリジナルゲーム**は現状「ライブラリから手動で適用」まで。将来的に `templateId` やゲーム名を指定する拡張で「〇〇（オリジナル）の会場作って」も可能になる想定。

## 5. 運用メモ

- **サーバー再起動**: ポート 3010 のプロセスを `taskkill` で終了後、`server` で `node index.js` を再起動。作業とセットで行うルールあり。
- **環境変数**: `server/.env` に `GEMINI_API_KEY`、`DATABASE_URL` 等。`client/.env.local` に `NEXT_PUBLIC_SOCKET_URL`（任意、未設定時は `window.location.origin` を使用）。

---

## 6. 今後の拡張（オリジナルゲームの会場作成）

- **現状**: 会場作成は「ミックスジュース / ババ抜き / 神経衰弱」の 3 モード＋キーワードフォールバックのみ。
- **拡張案**: `change_mode` に `templateId` またはゲーム名（タイトル）を追加し、サーバーで該当 `GameTemplate` を検索して適用。AI には「利用可能テンプレート一覧」をコンテキストで渡し、「〇〇の会場作って」でオリジナルも選べるようにする。
